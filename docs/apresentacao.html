1. Objetivo

Permitir que o usuario controle seus lancamentos pelo terminal:

  • Cadastrar despesas e receitas
  • Editar e excluir lancamentos por ID
  • Consultar relatorios por periodo
  • Gerar relatorio em PDF (HTML imprimivel)
  • Persistir tudo em PostgreSQL hospedado no Supabase

Foco: simplicidade, didatica e atender requisitos da disciplina (lambda, enum, ArrayList, Value Object).

6.1 Exemplo de Lambda

public BigDecimal totalDespesas(List<Movimento> movimentos) {
    return movimentos.stream()
            .filter(m -> m.getTipoMovimento().isDespesa()) // <-- LAMBDA
            .map(Movimento::getValor)
            .reduce(BigDecimal.ZERO, BigDecimal::add);
}

public long totalPendentes(List<Movimento> movimentos) {
    return movimentos.stream()
            .filter(m -> !m.isDebitado()) // <-- LAMBDA
            .count();
}

Filtragem, mapeamento e reducao via Stream API, sem for manual.

6.3 Value Object: PeriodoRelatorioVO

public final class PeriodoRelatorioVO {
    private final LocalDate dataInicial;
    private final LocalDate dataFinal;

    public PeriodoRelatorioVO(LocalDate ini, LocalDate fim) {
        if (ini == null || fim == null)
            throw new IllegalArgumentException("Datas obrigatorias.");
        if (ini.isAfter(fim))
            throw new IllegalArgumentException("Inicial > final.");
        this.dataInicial = ini;
        this.dataFinal   = fim;
    }
    // getters, equals, hashCode...
}

final imutavel equals/hashCode por valor

8. Validacoes & tratamento de erros

No service

  • Valor > 0
  • Descricao obrigatoria
  • Banco obrigatorio para Cartao/Pix
  • Parcelas >= 1

Na entrada (InputUtil)

  • Repete prompt ate o tipo ser valido
  • Datas em dd/MM/yyyy
  • Decimais com virgula ou ponto

Banco

  • PreparedStatement previne SQL Injection
  • try-with-resources evita vazamento de conexao
  • Erros reembalados com URL/usuario para diagnostico

UX

  • Mensagens prefixadas com >>
  • Confirmacao em operacoes destrutivas
  • Resumo do registro antes de editar/excluir
